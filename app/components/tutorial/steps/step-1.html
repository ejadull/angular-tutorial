Step 1: Create the directory structure

To start with step 1, please clone the repository from https://github.com/buccfer/angular-tutorial and fetch the step-1 branch. 

git fetch && git checkout step-1
Note: This tutorial assumes that all javascript and css files are imported in the index.html file (i.e: angular.min.js, bootstrap.min.css, etc). In this example we use Gulp.js along with NPM to build the project, but this is outside the scope of this tutorial. For more details, you can take a look at the gulpfile.js and the Gulp official site.

1. The first thing we need to do is to define the directory structure. There are many different ways to structure your application and each has it own benefits depending on the nature of the project. There are also many different generators such as Yeoman (see http://yeoman.io/generators/) that automatically create the directory structure and files for you, according to the best practices.

In this tutorial I will show a simple and practical way to structure your project that works for small client web applications.

                    app/
                    ----- shared/ All reusable components.
                    ---------- sidebar/
                    --------------- sidebarDirective.js
                    --------------- sidebarView.html
                    ---------- article/
                    --------------- articleDirective.js
                    --------------- articleView.html
                    ----- components/ Each component is treated as a mini Angular app.
                    ---------- home/
                    --------------- homeController.js
                    --------------- homeView.html
                    ---------- blog/
                    --------------- blogController.js
                    --------------- blogView.html
                    ----- app.module.js The main module of the AngularJs App.
                    assets/
                    ----- img/ Images and icons for your app.
                    ----- css/ All styles and style related files (SCSS or LESS files).
                    ----- js/  JavaScript files written for your app that are not for angular.
                    ----- libs/ Third-party libraries such as jQuery, Moment, Underscore, etc.
                    index.html The unique entire page of the SPA.

2. The second thing we need to do is to declare the scope of the Angular application in our index.html file as below.

index.html

                    <!DOCTYPE html>
                    <html lang="en" ng-app="videoApp"> Add ng-app to the DOM element that will be the root for your Angular app.
                        <head>                         IMPORTANT: The value of the ng-app must match with the app module name.
                            <meta charset="utf-8">
                            <meta http-equiv="X-UA-Compatible" content="IE=edge">
                            <meta name="viewport" content="width=device-width, initial-scale=1">
                            <meta name="description" content="AngularJS - Video App Tutorial">
                            <meta name="author" content="Fernando Buccella">
                            <title>Video App</title>

                            #All injected css files come here.. See gulp task 'inject' in gulpfile.js
                            #inject:css
                            #endinject

                            #HTML5 Shim and Respond.js IE8 support of HTML5 elements and media queries
                            #WARNING: Respond.js doesn't work if you view the page via file://
                            #[if lt IE 9]>
                            <script src="https://oss.maxcdn.com/libs/html5shiv/3.7.0/html5shiv.js"></script>
                            <script src="https://oss.maxcdn.com/libs/respond.js/1.4.2/respond.min.js"></script>
                            #<![endif]

                            <script src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/3.10.1/lodash.min.js"></script>
                        </head>
                        <body>
                            #All injected js files come here.. See gulp task 'inject' in gulpfile.js
                            #inject:js
                            #endinject
                        </body>
                    </html>
